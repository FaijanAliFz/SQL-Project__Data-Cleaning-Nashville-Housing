# 🧹 SQL Data Cleaning – Nashville Housing Dataset

---

## 📊 Project Overview
This project focuses on cleaning and transforming raw housing data using SQL. The dataset contained missing values, duplicate records, and inconsistent formats, which were processed into a structured and analysis-ready format.

---

## 🎯 Objectives
- Standardize date formats  
- Handle missing property address values  
- Split address fields into structured columns  
- Convert categorical values into readable format  
- Remove duplicate records  
- Drop unnecessary columns  

---

## ⚙️ Key Operations Performed
- Data type conversion for date fields  
- Handling null values using self joins  
- Splitting address columns into address, city, and state  
- Standardizing categorical values (Y/N → Yes/No)  
- Identifying and removing duplicate records  
- Dropping irrelevant columns  

---

## 🛠️ SQL Techniques Used
- Joins (Self Join)  
- Window Functions (ROW_NUMBER)  
- Common Table Expressions (CTE)  
- String Functions (SUBSTRING, CHARINDEX, PARSENAME)  
- CASE Statements  
- Data Type Conversion  

---

## 📈 Key Outcomes
- Improved data consistency and accuracy  
- Removed duplicate and incomplete records  
- Structured unorganized address data  
- Prepared dataset for further analysis and visualization  

---

## 🚀 Project Outcome
Transformed raw data into a **clean, structured, and analysis-ready dataset** using SQL, enabling more reliable insights.
